# 프로젝트 기획서 (PoC / 프로토타입)

> 목표 : NPC와의 대면, 상호작용, 정산, 장비 대여, 관계 변화, 사건 발생이 실제 플레이에서 재미있는지 검증한다.

---

# 1. 게임 개요

| 항목 | 내용 |
|---|---|
| 장르 | 생존 / 전략 / 파티 운영 |
| 핵심 키워드 | NPC 대면, 신뢰와 배신, 정산, 기력 관리, 장비 대여, 심리전 |
| 플레이 방식 | 탐색 + 원거리 공격 + 우클릭 상호작용 + 상황 로그/말풍선 |
| 핵심 목표 | NPC들과 관계를 유지하거나 끊어가며 생존하기 |

---

# 2. 핵심 컨셉

> “NPC는 단순 전투 유닛이 아니라, 각자의 욕망과 두려움을 가진 존재다.”

플레이어는 탐색 중 NPC를 만나고, 멀리서 말을 걸거나, 영입하거나, 무시하거나, 공격할 수 있다.

NPC는 각자의 성향에 따라:

- 플레이어를 경계하고
- 장비나 전리품에 관심을 보이고
- 정산을 요구하고
- 위험하면 도망치고
- 불만이 쌓이면 떠나거나 배신한다.

이 게임의 핵심은 전투 자체보다, **사람을 만나고 읽고 의심하며 운영하는 과정**에 있다.

---

# 3. 핵심 루프

```text
탐색
 ↓
NPC 대면
 ↓
상호작용 / 공격 / 무시
 ↓
관계 변화
 ↓
정산 / 장비 대여 / 휴식
 ↓
떠남 / 도주 / 배신 / 협력
 ↓
탐색
```

---

# 4. 기본 조작

## 4-1. 이동

```text
WASD 이동
```

---

## 4-2. 공격

```text
좌클릭 = 원거리 공격
```

PoC에서는 모든 공격을 원거리 공격으로 처리한다.

### 이유

- NPC와 거리를 둔 상태에서도 전투 가능
- “가까이 오지 마”라고 말하는 NPC를 억지로 근접 전투에 넣지 않아도 됨
- 공격과 상호작용을 명확히 구분 가능
- PoC 전투 구현이 단순해짐

---

## 4-3. 상호작용

```text
우클릭 = 대상 상호작용
```

플레이어는 일정 거리 안에 있는 NPC나 동료를 우클릭해 상호작용할 수 있다.

```text
InteractionRange = 6.0f
```

상호작용 거리 안에 있으면 대상 위에 표시가 뜬다.

```text
우클릭 : 상호작용
```

---

# 5. 상호작용 구조

## 5-1. 비동료 NPC 상호작용

비동료 NPC에게 우클릭하면 상호작용 메뉴가 열린다.

```text
[영입 제안]
[상태 확인]
[무시]
```

공격은 메뉴에 넣지 않는다.

```text
공격 = 좌클릭
상호작용 = 우클릭
```

---

## 5-2. 동료 NPC 상호작용

동료에게 우클릭하면 동료 메뉴가 열린다.

```text
[정산하기]
[장비 대여]
[장비 회수]
[상태 확인]
[대기 지시]
[파티에서 내보내기]
```

# 6. 캐릭터 스탯

## 6-1. 플레이어 스탯

| 스탯 | 설명 |
|---|---|
| HP | 체력 |
| Stamina | 기력 |
| ATK | 공격력 |
| Threat | 위압감 |
| Wealth | 재력 노출도 |

플레이어에게 Morality는 두지 않는다.

플레이어의 선악을 시스템이 직접 규정하지 않고, NPC가 각자의 성향에 따라 플레이어의 행동을 해석하게 한다.

---

## 6-2. NPC 스탯

| 스탯 | 설명 |
|---|---|
| HP | 체력 |
| Stamina | 기력 |
| ATK | 공격력 |
| Trust | 플레이어에 대한 신뢰 |
| Greed | 탐욕 |
| Fear | 겁 / 생존 본능 |
| Morality | 선악 성향 |

# 7. 장비 시스템

## 7-1. 장비의 역할

장비는 단순 보상이 아니라 **파티 전력 배치 수단**이다.

장비는 NPC에게 선물하는 개념이 아니라, 플레이어가 임시로 맡기는 **대여** 개념으로 처리한다.

```text
장비 = 플레이어 소유의 파티 자원
정산 = 동료가 받아야 할 보상
```

따라서 장비 대여와 정산은 분리한다.

---

## 7-2. 장비 종류

PoC에서는 3종류만 사용한다.

| 종류 | 역할 |
|---|---|
| 무기 | ATK 중심 |
| 갑옷 | HP 또는 Threat 중심 |
| 장신구 | Wealth 또는 보조 수치 중심 |

---

## 7-3. 장비 스탯

모든 장비는 다음 수치를 가질 수 있다.

| 스탯 | 설명 |
|---|---|
| ATK | 공격력 증가 |
| HP | 체력 증가 |
| Threat | 위압감 증가 |
| Wealth | 재력 노출도 증가 |
| Value | 정산 계산에 사용되는 가치 |

---

## 7-4. 최종 스탯 계산

```text
최종 HP = 기본 HP + 착용 장비 HP 총합
최종 ATK = 기본 ATK + 착용 장비 ATK 총합
Threat = 기본 Threat + 착용 장비 Threat 총합 + 전투 상태 보정
Wealth = 착용 장비 Wealth 총합
```

---

## 7-5. NPC 인벤토리

각 NPC는 별도 인벤토리를 가진다.

플레이어가 NPC에게 장비를 대여하면 해당 장비는 NPC 인벤토리에 들어간다.

```text
플레이어 인벤토리 → NPC 인벤토리
```

NPC는 자신의 인벤토리 안에서 최적 장비를 자동으로 착용한다.

---

## 7-6. 장비 슬롯

PoC에서는 슬롯을 단순하게 유지한다.

```text
무기 1개
갑옷 1개
장신구 1개
```

---

## 7-7. NPC 자동 착용

NPC는 인벤토리 안의 장비 중 현재 자신에게 가장 적합한 장비를 자동으로 착용한다.

착용 기준은 NPC 성향에 따라 다르다.

```text
장비 점수 =
ATK * ATK가중치
+ HP * HP가중치
+ Threat * Threat가중치
+ Wealth * Wealth가중치
```

---

## 7-8. 성향별 장비 선호

### Fear 높은 NPC

생존을 중시한다.

```text
장비 점수 =
ATK * 0.8
+ HP * 1.5
+ Threat * 0.7
+ Wealth * 0.2
```

---

### Greed 높은 NPC

비싸 보이는 장비에 끌린다.

```text
장비 점수 =
ATK * 1.0
+ HP * 0.8
+ Threat * 0.5
+ Wealth * 1.4
```

---

### Morality 낮은 NPC

자기 이득과 기회주의를 중시한다.

```text
장비 점수 =
ATK * 1.3
+ HP * 0.7
+ Threat * 1.1
+ Wealth * 0.8
```

---

### 기본형 NPC

균형 있게 판단한다.

```text
장비 점수 =
ATK * 1.0
+ HP * 1.0
+ Threat * 0.7
+ Wealth * 0.4
```

---

## 7-9. 장비 대여

장비 대여는 기본적으로 Trust에 직접 영향을 주지 않는다.

```text
Trust 변화 없음
```

다만 NPC는 상황에 따라 말풍선이나 로그로 반응할 수 있다.

```text
“일단 맡아두지.”
“필요하면 쓰겠어.”
“이건 꽤 괜찮은데?”
```

---

## 7-10. 장비 회수

장비 회수도 기본적으로 Trust에 직접 영향을 주지 않는다.

```text
Trust 변화 없음
```

다만 특정 상황에서는 불만이나 불안을 유발할 수 있다.

### 불만/불안 조건

```text
회수 장비가 현재 착용 중이었음
전투 직전 또는 위험 상황
NPC Trust 낮음
NPC Greed 높음
NPC Stamina 낮음
```

이 경우 Trust를 직접 깎기보다는 임시 불만 수치 또는 탈퇴 점수에 소폭 반영한다.

```text
탈퇴 점수 +5
또는
불만 상태 플래그 부여
```

말풍선 예시:

```text
“지금 가져간다고?”
“그거 없으면 좀 불안한데.”
“아쉽네. 쓸 만했는데.”
```

---

## 7-11. 장비와 이탈

NPC가 파티를 떠나거나 배신할 경우, NPC 인벤토리에 있는 장비 처리가 중요하다.

### 자진 탈퇴

```text
Trust 높음 → 장비 반환 후 떠남
Trust 낮음 → 일부 또는 전체 장비를 가지고 떠남
```

### 도주

```text
현재 착용 장비를 들고 도망
NPC 인벤토리 장비도 함께 손실 가능
```

### 배신

```text
대여 장비를 착용한 상태로 적대
플레이어가 투자한 전력이 위협으로 전환됨
```

# 8. 전리품과 정산 시스템

## 8-1. 정산의 역할

정산은 장비 대여와 다르다.

```text
정산 = 동료가 받아야 할 몫에 대한 보상
장비 대여 = 파티 전력 배치를 위한 임시 자원 이동
```

장비는 대여 개념이므로, 정산은 기본적으로 **골드 지급** 중심으로 처리한다.

다만 획득 장비의 Value는 미정산금 계산에 포함된다.

---

## 8-2. 전리품 총 가치

```text
전리품 총 가치 = 획득 골드 + 획득 장비 Value 총합
```

예시:

```text
획득 골드 = 500
획득 장비 = Value 300 무기 1개

전리품 총 가치 = 800
```

---

## 8-3. 분배 대상

전리품 획득 시점에 파티에 속한 캐릭터가 분배 대상이다.

```text
분배 인원 = 플레이어 + 현재 동료 수
```

---

## 8-4. 동료별 미정산금 계산

```text
동료 1인당 받을 몫 = 전리품 총 가치 / 분배 인원
```

예시:

```text
플레이어 + 동료 1명
전리품 총 가치 = 800

1인당 몫 = 800 / 2 = 400

동료 A 미정산금 += 400
```

동료가 나중에 합류한 경우, 합류 이후 획득한 전리품부터 정산 대상이 된다.

---

## 8-5. 정산 UI

동료에게 우클릭 후 `[정산하기]`를 선택하면 정산 UI가 열린다.

```text
현재 미정산금 : 400
지급 골드 입력 : [    ]
[정산하기]
[취소]
```

PoC에서는 장비 지급을 정산에 포함하지 않는다.

장비는 대여 시스템에서 따로 처리한다.

---

## 8-6. 정산 비율

```text
정산 비율 = 지급 골드 / 미정산금
```

---

## 8-7. Trust 변화 규칙

```text
80% ~ 100% = 허용 범위
100% 초과 = 후한 정산
80% 미만 = 부족 정산
0% = 미지급
```

---

## 8-8. Trust 변화 계산식

```text
정산 비율 = 지급 골드 / 미정산금

if 정산 비율 == 0:
    Trust 변화 = -15

else if 정산 비율 < 0.8:
    부족률 = 0.8 - 정산 비율
    Trust 변화 = -5 - (부족률 * 25)

else if 정산 비율 <= 1.0:
    Trust 변화 = +3

else:
    초과율 = 정산 비율 - 1.0
    Trust 변화 = +3 + min(초과율 * 20, 10)
```

---

## 8-9. Greed 보정

Greed가 높은 NPC는 정산에 더 민감하다.

```text
Greed 보정 = 1 + ((Greed - 50) / 100)
```

보정 범위:

```text
최소 0.7
최대 1.5
```

최종 Trust 변화:

```text
최종 Trust 변화 = 기본 Trust 변화 * Greed 보정
```

---

## 8-10. 정산 후 미정산금 처리

```text
미정산금 = max(0, 미정산금 - 지급 골드)
```

초과 지급분은 다음 정산으로 이월하지 않는다.

# 9. 동료 영입 시스템

## 9-1. 영입 방식

비동료 NPC를 우클릭한 뒤 `[영입 제안]`을 선택하면 영입 판정을 한다.

영입은 확정이 아니라 점수 기반으로 성공/거절된다.

---

## 9-2. 영입 점수

```text
영입 점수 =
50
+ (플레이어 Threat * 0.2)
+ (플레이어 Wealth * 0.1)
- (NPC Fear * 0.3)
+ (NPC Greed * 0.1)
+ ((NPC Morality - 50) * 0.1)
```

---

## 9-3. 영입 성공 기준

```text
영입 점수 >= 50
→ 영입 가능

영입 점수 < 50
→ 영입 거절
```

---

## 9-4. 영입 성공 시

```text
NPC가 파티에 합류
미정산금 0으로 시작
NPC 인벤토리 유지
초기 Trust 설정
```

초기 Trust:

```text
초기 Trust = max(NPC 기본 Trust, 40 + (영입 점수 - 50) * 0.2)
```

---

## 9-5. 영입 거절 반응

### Fear 높은 NPC

```text
“난 너랑 못 가.”
“위험해 보여.”
```

### Greed 높은 NPC

```text
“조건이 별로인데?”
“내 몫은 확실한 거지?”
```

### Morality 높은 NPC

```text
“당신을 아직 믿을 수 없군.”
```

# 10. 파티 이탈 시스템

동료가 파티에서 나가는 방식은 4가지다.

```text
플레이어가 내보냄
NPC가 자진 탈퇴
NPC가 전투 중 도주
NPC가 배신
```

---

# 10-1. 플레이어가 내보냄

동료 우클릭 후 `[파티에서 내보내기]`를 선택한다.

선택지:

```text
[정산 후 내보내기]
[그냥 내보내기]
```

장비 회수는 별도 메뉴에서 먼저 처리한다.

---

## 정산 후 내보내기

```text
정산 UI 실행
정산 결과 반영
NPC 파티 탈퇴
```

Trust가 높으면 나중에 다시 만났을 때 적대하지 않는다.

---

## 그냥 내보내기

미정산금이 남아 있으면 Trust가 하락한다.

```text
Trust 변화 = -5 - (미정산금 / 100)
```

---

# 10-2. NPC 자진 탈퇴

NPC가 플레이어와 더 이상 함께할 수 없다고 판단하면 자진 탈퇴한다.

배신이 아니라 관계 실패다.

---

## 탈퇴 점수

```text
탈퇴 점수 =
(100 - Trust)
+ 미정산 불만
+ Stamina 부족 보정
+ 최근 불만 행동 보정
```

세부 보정:

```text
미정산 불만 = 미정산금 / 20
Stamina 부족 보정 = Stamina 30 이하일 때 +15
최근 불만 행동 보정 = +10
```

최근 불만 행동 예시:

```text
정산 거부
기력 낮은 상태로 계속 이동
위험 상황 방치
전투 직전 핵심 장비 회수
```

---

## 탈퇴 기준

```text
탈퇴 점수 >= 80
→ 탈퇴 경고

탈퇴 점수 >= 100
→ 자진 탈퇴
```

---

# 10-3. 전투 중 도주

도주는 Fear 중심 행동이다.

NPC가 플레이어를 해치려는 것이 아니라, 자기 생존을 우선해 도망치는 것이다.

---

## 도주 점수

```text
도주 점수 =
NPC Fear
+ 전투 불리 보정
+ NPC HP 부족 보정
+ NPC Stamina 부족 보정
- Trust 보정
```

세부 보정:

```text
전투 불리 보정:
적 수가 아군보다 많으면 +15
적 평균 Threat가 아군보다 높으면 +15

NPC HP 부족 보정:
HP가 30% 이하이면 +20

NPC Stamina 부족 보정:
Stamina가 20 이하이면 +15

Trust 보정:
Trust * 0.3 만큼 감소
```

---

## 도주 기준

```text
도주 점수 >= 90
→ 도주 준비

도주 점수 >= 110
→ 전투 중 도주
```

---

# 10-4. 배신

배신은 플레이어에게 직접 피해를 주는 적대 행동이다.

Fear가 높은 NPC는 정면 배신보다 도주 쪽으로 행동해야 한다.

---

## 배신 점수

```text
배신 점수 =
(100 - Trust)
+ Greed
+ (100 - Morality)
+ 플레이어 Wealth
+ 플레이어 약화 보정
- 플레이어 Threat
- Fear 보정
```

세부 보정:

```text
플레이어 HP 30% 이하이면 +20
플레이어 Stamina 20 이하이면 +15
Fear 보정 = NPC Fear * 0.3
```

---

## 배신 기준

```text
배신 점수 >= 120
→ 배신 징조

배신 점수 >= 140
→ 배신 발생
```

PoC 배신 종류:

```text
휴식 중 절도
전투 중 플레이어 공격
```

# 11. 기력과 휴식

## 11-1. 기력 감소

| 상황 | 감소량 |
|---|---:|
| 시간 경과 | -1 / 10초 |
| 원거리 공격 | -1 |
| 전투 참여 | -5 |
| 피격 | -3 |

---

## 11-2. 기력 상태

| Stamina | 상태 | 효과 |
|---:|---|---|
| 70~100 | 정상 | 효과 없음 |
| 40~69 | 피로 | 공격 주기 소폭 증가 |
| 20~39 | 지침 | ATK 20% 감소 |
| 1~19 | 탈진 직전 | ATK 40% 감소, 도주/탈퇴 위험 증가 |
| 0 | 탈진 | 일정 시간 행동 제한 |

---

## 11-3. 휴식

휴식은 전원에게 적용된다.

```text
휴식 시간 = 5초
HP +30%
Stamina +70
```

휴식 후 각 동료에 대해 이벤트 판정을 한다.

```text
1. 배신 점수 확인
2. 탈퇴 점수 확인
3. 요구 점수 확인
4. 무사 휴식
```

# 12. 요구 시스템

NPC는 상황에 따라 먼저 요구할 수 있다.

## 12-1. 요구 종류

```text
정산 요구
휴식 요구
거리 유지 요구
```

장비 요구는 PoC에서는 제외한다.

이유:

```text
장비는 선물이 아니라 대여 자원이기 때문에
NPC가 소유권을 요구하는 구조와 맞지 않음
```

---

## 12-2. 정산 요구 점수

```text
정산 요구 점수 =
Greed
+ (미정산금 / 10)
+ 플레이어 Wealth
- Trust
```

기준:

```text
정산 요구 점수 >= 80
→ 로그 또는 말풍선 출력

정산 요구 점수 >= 100
→ 직접 정산 요구 이벤트 발생
```

# 13. NPC 반응 시스템

## 13-1. 기본 방향

상황 로그와 말풍선은 중앙 시스템이 주기적으로 뿌리는 것이 아니라, **각 NPC가 자신의 상태 변화에 따라 개별적으로 발생**시킨다.

```text
NPC가 반응을 결정한다.
LogManager/BubbleManager는 표시만 담당한다.
```

---

# 13-2. 로그와 말풍선 역할 구분

## 상황 로그

플레이어가 관찰한 사실.

```text
정찬서가 플레이어의 장비를 힐끗거렸다.
박민호가 한 걸음 뒤로 물러났다.
유윤상이 말없이 주변을 살핀다.
```

역할:

```text
징조 전달
상태 암시
사건 예고
판단 재료 제공
```

---

## 말풍선

NPC가 직접 말하는 대사.

```text
“거기서 멈춰.”
“내 몫은?”
“난 못 해!”
```

역할:

```text
즉각 반응
의사 표현
캐릭터성 표현
```

---

# 13-3. 반응 발생 구조

반응은 단순 주기 출력이 아니라 다음 구조로 발생한다.

```text
상태 변화 발생
 ↓
NPC 반응 후보 생성
 ↓
확률 판정
 ↓
쿨타임 확인
 ↓
로그 또는 말풍선 출력
```

---

# 13-4. 상태 변화 트리거

다음 상황에서 NPC는 반응 후보를 만든다.

```text
Trust가 특정 구간 이하로 내려감
미정산금이 일정 이상 쌓임
플레이어 Wealth가 높아짐
플레이어 Threat가 높아짐
NPC Stamina가 낮아짐
전투가 불리해짐
휴식 시작
장비 대여 발생
장비 회수 발생
도주 점수 상승
배신 점수 상승
탈퇴 점수 상승
```

---

# 13-5. 확률적 출력

트리거가 발생해도 항상 출력하지 않는다.

조건을 만족하면 출력 확률을 계산한다.

```text
조건 만족
→ 출력 확률 계산
→ 성공 시 로그 또는 말풍선 출력
```

예시:

```text
Greed 90 NPC가 Wealth 70 플레이어를 봄
→ 높은 확률로 반응

Greed 40 NPC가 Wealth 70 플레이어를 봄
→ 낮은 확률로 반응
```

---

# 13-6. 쿨타임

반응이 너무 자주 나오지 않도록 쿨타임을 둔다.

```text
NPC별 상황 로그 쿨타임 = 15초
NPC별 말풍선 쿨타임 = 7초
동일 카테고리 반복 쿨타임 = 30초
```

---

# 13-7. 보조적 유지 반응

완전히 주기적으로 출력하지는 않는다.

다만 위험 상태가 오래 유지되면 낮은 확률로 다시 알려줄 수 있다.

```text
위험 상태 유지 중
→ 20~30초마다 낮은 확률로 재출력
```

예시:

```text
Trust 낮은 상태가 오래 지속됨
→ 가끔 “유윤상이 말없이 거리를 둔다.” 출력
```

---

# 14. 반응 예시

출력 확률 =

정찬서가 플레이어의 장비를 힐끗거렸다.

조건:

```text
NPC Fear >= 70
플레이어 Threat >= 50
```

확률:

출력 확률 =
20%
+ (Fear - 70) * 1%

로그:

박민호가 플레이어와 거리를 벌렸다.

로그:

유윤상이 대답을 피한다.

## 14-6. 장비 회수

장비를 NPC에게서 회수했을 때.

기본적으로 Trust 변화 없음.

조건에 따라 불만/불안 반응 출력.

로그:

```text
정찬서가 아쉬운 듯 장비를 바라본다.
박민호가 불안한 표정을 짓는다.
```

말풍선:

```text
“지금 가져간다고?”
“그거 없으면 좀 불안한데.”
```

# 15. 전투 시스템

PoC 전투는 단순하게 구성한다.

목적은 액션성이 아니라, 전투 상황에서 NPC 성향이 드러나는지 확인하는 것이다.

```text
플레이어 + 동료
vs
적
```

- 실시간 전투
- 원거리 공격 기반
- 플레이어 좌클릭 공격
- 동료 자동 전투

---

## 15-1. NPC 전투 행동

| 조건 | 행동 |
|---|---|
| Trust 높음 | 플레이어 근처 유지, 적극 전투 |
| Fear 높음 | 위험하면 후퇴, 도주 가능 |
| Greed 높음 | 전투 후 전리품/정산 요구 |
| Morality 낮음 | 자기 생존과 이득 우선 |

# 18. PoC 검증 목표

| 목표 | 검증 내용 |
|---|---|
| NPC가 살아있는 존재처럼 느껴지는가 | 로그/말풍선/행동 차이 확인 |
| 정산이 의미 있는 선택인가 | 미정산, 지급량, Trust 변화 확인 |
| 장비 대여가 전력 운영처럼 느껴지는가 | NPC 자동 착용과 회수 처리 확인 |
| 떠남/도주/배신이 구분되는가 | 플레이어가 원인을 이해하는지 확인 |
| Fear가 배신이 아닌 도주로 느껴지는가 | 겁 많은 NPC 행동 확인 |
| 로그/말풍선이 자연스러운가 | 스팸 없이 징조 전달이 되는지 확인 |
